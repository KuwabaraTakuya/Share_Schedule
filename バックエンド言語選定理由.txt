■ 結論: Go (Golang) を選定する

--------------------------------------------------------------------------------
【比較対象言語と評価軸】
--------------------------------------------------------------------------------

評価軸:
  1. 実行速度・レイテンシ
  2. メモリ効率（ホスティングコスト）
  3. 並行処理能力（チャット接続の同時処理）
  4. コールドスタート時間（Cloud Run など）
  5. 開発生産性
  6. Firebase / Google Cloud エコシステムとの相性
  7. 将来的な保守性

--------------------------------------------------------------------------------
【言語別評価】
--------------------------------------------------------------------------------

▼ Go (Golang) ★採用
─────────────────────────────────
実行速度     : ★★★★★  コンパイル済みバイナリで C に近い速度
メモリ効率   : ★★★★★  アイドル時 10〜30MB 程度。非常に軽量
並行処理     : ★★★★★  goroutine (軽量スレッド) が標準搭載。
                       数万の同時接続を低コストで処理可能
コールドスタート: ★★★★★  Cloud Run での起動が 100ms 以下
開発生産性   : ★★★☆☆  静的型付けで安全だが Node.js より冗長な部分あり
Google Cloud  : ★★★★★  Google 製言語のため Cloud Run, Firestore SDK,
                       Gemini SDK すべて公式サポート
保守性       : ★★★★☆  シンプルな文法で読みやすく長期保守に向く

具体的なベンチマーク参考値（小規模サーバー, 1 vCPU / 512MB RAM）:
  - HTTP リクエスト処理: 約 50,000〜80,000 req/s
  - WebSocket 同時接続: 数万接続を 30MB 程度のメモリで処理可能
  - コールドスタート (Cloud Run): 50〜200ms

採用理由の詳細:
  1. 本アプリはリアルタイムチャット・カレンダー同期・Gemini API 呼び出しなど
     I/O バウンドかつ並行処理が多い。goroutine はこの用途に最適。
  2. Firebase Firestore はリアルタイム通信を担うため、バックエンドは
     ビジネスロジック（LLM 連携・ファイル検証・カレンダー解析）に集中できる。
     Go はこの処理を高速・低コストで実行できる。
  3. Google Cloud Run + Go の組み合わせはコールドスタートが最速クラスであり、
     少人数コミュニティ向けのトラフィックパターン（バーストあり・普段は少ない）
     に非常に相性が良い。スケールゼロ時のコストが最小化できる。
  4. Gemini API, Google Calendar API, Google Maps API など Google 系 API は
     Go の公式 SDK が充実しており、認証 (Application Default Credentials) も
     Cloud 環境でシームレスに統合できる。
  5. 静的型付けにより大規模になっても型エラーをコンパイル時に検出でき、
     セキュリティ上のバグを未然に防ぎやすい。

─────────────────────────────────

▼ Node.js (TypeScript)
─────────────────────────────────
実行速度     : ★★★☆☆  V8 JIT で高速だが Go に比べ 3〜5 倍遅い
メモリ効率   : ★★☆☆☆  起動時から 100〜200MB 消費。V8 ヒープが大きい
並行処理     : ★★★☆☆  シングルスレッド (Event Loop)。CPU バウンド処理は
                       Worker Threads が必要で複雑になる
コールドスタート: ★★★☆☆  Cloud Run での起動 500ms〜2s (依存パッケージが大きいほど遅い)
開発生産性   : ★★★★★  フロントエンドと同じ TypeScript で書けるため
                       開発者体験は最高。npm エコシステムも豊富
Google Cloud  : ★★★★☆  Firebase SDK (公式) との親和性は最高
保守性       : ★★★★☆  TypeScript 採用で型安全性は確保できる

不採用理由:
  - チャット系アプリでは多数の同時接続を並行処理する場面があるが、
    Node.js のシングルスレッド+イベントループ方式では Go の goroutine に比べ
    スループットが低い。
  - メモリ消費が多く、Cloud Run のコストが Go より高くなる。
  - フロントエンドが React (TypeScript) なので言語統一のメリットはあるが、
    バックエンドの性能面でトレードオフが大きすぎると判断。

─────────────────────────────────

▼ Python (FastAPI / Django)
─────────────────────────────────
実行速度     : ★★☆☆☆  インタープリタ型のため最も遅い。Go の 10〜20 倍遅い
メモリ効率   : ★★☆☆☆  Go より多く消費（150〜300MB）
並行処理     : ★★★☆☆  asyncio で I/O 非同期は可能だが GIL の制約あり
コールドスタート: ★★☆☆☆  大きな依存パッケージで 1〜5s
開発生産性   : ★★★★★  最も書きやすい。ML ライブラリ豊富
Google Cloud  : ★★★★☆  公式 SDK あり

不採用理由:
  - 本アプリのバックエンドで機械学習のモデルをホストするわけではなく、
    LLM は Gemini API を外部呼び出しするだけなので Python の ML 優位性が活かせない。
  - 速度・メモリ面で最も不利。少人数向けとはいえ、チャットアプリの
    レスポンシブネスは重要であり、Python では要件を満たすのが難しい。

─────────────────────────────────

▼ Rust
─────────────────────────────────
実行速度     : ★★★★★  Go 以上の速度（場合によっては C 並み）
メモリ効率   : ★★★★★  最小限のメモリ使用量
並行処理     : ★★★★★  async/await + Tokio ランタイムで高並行処理可能
コールドスタート: ★★★★★  最速
開発生産性   : ★☆☆☆☆  学習コストが非常に高い（所有権・借用・ライフタイム）
Google Cloud  : ★★★☆☆  公式 SDK は限定的。コミュニティ製を使うことになる
保守性       : ★★★☆☆  高品質なコードになるが、書ける人が限られる

不採用理由:
  - 開発速度が大幅に遅くなる。本プロジェクトは機能の実装と反復を重視するため
    Rust の学習・開発コストは過剰投資となる。
  - Go で十分な性能（数万 req/s）が出せるため、Rust を使う性能上の必然性がない。
  - Google 公式の Go SDK と比べ Rust の Google API ライブラリは成熟度が低い。

─────────────────────────────────

▼ Bun (JavaScript/TypeScript runtime)
─────────────────────────────────
実行速度     : ★★★★☆  Node.js より 3〜5 倍高速（Zig ベース）
メモリ効率   : ★★★★☆  Node.js より効率的
開発生産性   : ★★★★★  Node.js と同様に TypeScript で書ける
Google Cloud  : ★★★☆☆  Cloud Run でコンテナ実行は可能
保守性       : ★★★☆☆  まだ新しく、本番環境での採用実績が少ない

不採用理由:
  - 2024〜2026 年時点でエコシステムの成熟度が低く、本番プロダクションへの
    採用リスクが高い。
  - Firebase Admin SDK の Bun 対応が Node.js ほど安定していない。

【Go の採用構成詳細】
--------------------------------------------------------------------------------

フレームワーク: Fiber v2
  - Go の HTTP フレームワークで最速クラス
  - Express.js に似た API 設計で直感的
  - ミドルウェアエコシステムが充実

デプロイ: Google Cloud Run
  - コンテナベースで自動スケーリング
  - スケールゼロ対応（コスト最適）
  - Firebase と同じ Google エコシステム

主なライブラリ:
  - firebase-admin-go    : Firestore/Auth/Storage 操作
  - google-api-go-client : Calendar API / Maps API
  - google/generative-ai-go : Gemini API
  - golang-jwt/jwt       : JWT 検証
  - go-chi/cors          : CORS 設定
